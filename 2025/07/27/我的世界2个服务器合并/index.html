我的世界2个服务器合并

起应：

今天我突发奇想，决定尝试将两个不同地图的服务器结合起来。特别的是，这两个服务器分别使用了Geyser和Floodgate插件，它们的网络配置稍有不同。由于目前网络上没有相关教程，我打算自己动手摸索一下，看看能不能成功。

第一步

安装 Geyser 和 Floodgate：

  1. Geyser：是一个用于将 Java 版 Minecraft 服务器和 Bedrock 版 Minecraft 服务器连接的插件https://geysermc.org/
  2. Floodgate：是 Geyser 的附加插件，用于允许 Bedrock 版玩家访问 Java 版服务器https://github.com/GeyserMC/Floodgate
  3. 设置 Velocity 作为代理，下载https://www.velocitypowered.com/

设置 Velocity 作为代理

  1. 服务器创建 Velocity 目录在你的计算机上选择一个位置（比如桌面或专门的文件夹），创建一个名为 Velocity 的文件夹。
  2. 将下载的 velocity-.jar 文件放入这个文件夹。
  3. 启动 Velocity：
  1. 在你刚刚创建的文件夹里面有velocity.toml这个文件，然后编辑他
    在 velocity.toml 文件中，检查以下配置项，确保代理服务器监听的端口正确设置。默认情况下，Velocity 监听在 25577 端口。如果你希望 Velocity 使用其他端口，可以修改此设置。
  1. 启用玩家信息转发
    如果你希望启用玩家信息转发（例如玩家的 UUID 和皮肤信息正确显示），可以找到并配置以下选项：

modern可以改为
“none”：不转发玩家信息。
“legacy”：使用较老的 UUID 格式进行转发。
“modern”：现代的 UUID 格式进行转发，建议使用此选项
6. 编辑 config.yml 文件（如果没有，创建一个）
